# Installation

VxLLM can be installed in several ways depending on your use case.

## Quick Install (Recommended)

Install VxLLM with a single command:

```bash
curl -fsSL https://vxllm.com/install.sh | sh
```

This script automatically:
- Detects your OS and architecture
- Installs [Bun](https://bun.sh/) if not present
- Clones and sets up VxLLM
- Creates the `vxllm` command in your PATH

After installation, pull a model and start chatting:

```bash
vxllm pull qwen2.5:7b
vxllm serve
```

## Manual CLI Install

If you already have Bun installed, you can install from source:

```bash
git clone https://github.com/vxllm/vxllm.git ~/.vxllm
cd ~/.vxllm
bun install
```

Then add the CLI to your PATH:

```bash
ln -sf ~/.vxllm/apps/cli/src/index.ts ~/.local/bin/vxllm
```

### Requirements

- [Bun](https://bun.sh/) v1.3 or later
- macOS (Apple Silicon or Intel), Linux, or Windows
- 8 GB RAM minimum (16 GB recommended for 7B models)

The desktop app bundles the server and voice service. No additional setup required.

## Docker

Pull from Docker Hub and run:

```bash
docker pull vxllm/vxllm
docker run -p 11500:11500 vxllm/vxllm
```

Or use Docker Compose for the full stack (server + voice service):

```bash
git clone https://github.com/vxllm/vxllm.git
cd vxllm

# Copy and configure environment
cp docker/.env.example docker/.env

# Start server and voice service
docker compose -f docker/docker-compose.yml up -d
```

The server will be available at `http://localhost:11500`.

See [Docker Deployment](/docs/deployment/docker) for full configuration options.

## Building from Source

```bash
git clone https://github.com/vxllm/vxllm.git
cd vxllm
bun install
bun run db:push
bun run dev:server
```

This starts the development server on `http://localhost:11500`.
